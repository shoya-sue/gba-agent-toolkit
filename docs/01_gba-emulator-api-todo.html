自前吸い出しROM対応・ローカルLLMエージェント連携ツール — TODOまとめ
結論

技術的に実現可能。GBAは mcp-mgba / mgba-mcp、GB/GBCは PyBoy を土台にすれば新規開発はほぼ不要。 残る作業は「環境構築 → 動作確認 → 自分用ランチャーGUI実装 → API拡張」というTODOの消化のみで、詳細設計はClaude Opusとの推論セッションに引き継ぐ。

本ツールの利用主体は人間ではなくAI Agent（将来的にローカルLLM）であるという前提が全体設計の核。 GUIの「ROM選択→Start」は人間による初期セットアップのみを担い、実際のプレイ操作・画面認識・状況判断はすべてAPI経由でAI Agentが行う。 そのためTODOの各項目は「人間が遊ぶための機能」ではなく「AI Agentが自律的に画面を見て・状況を読み取り・入力を送るためのインターフェース」として設計されている。
